# Análisis de Métricas Ágiles - UniScheduler

## 1. Evolución del proyecto (Burnup Chart)
El proyecto muestra un progreso constante desde el Sprint 0 hasta el Sprint 3. El trabajo completado (línea verde) se mantuvo cercano al trabajo planificado (línea azul), con una desviación mínima en el Sprint 2 debido a problemas en la validación de prerrequisitos.

## 2. Cuellos de botella (Control Chart)
El Sprint 3 presentó el tiempo de ciclo más alto (promedio 3.2 días por tarea) debido a la complejidad del algoritmo CSP. La tarea HU-011 (backtracking) tomó 5 días, identificándose como el principal cuello de botella. Se mitigó implementando pair programming.

## 3. Estabilidad del equipo (Velocity Chart)
La velocidad promedio del equipo fue de 29 Story Points por sprint, con una desviación estándar de 3.2 SP. Esto indica un equipo estable y predecible. El Sprint 3 tuvo la velocidad más alta (44 SP) debido a la concentración de tareas complejas.

## 4. Conclusión
El equipo demostró capacidad para entregar valor de manera consistente. Los cuellos de botella fueron identificados y mitigados a tiempo, permitiendo cumplir con el cronograma establecido.
