Este documento establece los criterios y escalas utilizados para la gestión de riesgos y oportunidades dentro del proyecto, así como las definiciones de los conceptos clave aplicados en la documentación.

---

## 1. Identificación y Escalas de Impacto

| Campo | Definición / Criterio |
|:---|:---|
| **ID Riesgo (Amenaza)** | Formato `R-XXX` (Ej: R-001, R-002). Nomenclatura secuencial y fácil de rastrear en la matriz de riesgos. |
| **ID Oportunidad** | Formato `OP-XXX` (Ej: OP-001, OP-002). Nomenclatura secuencial para diferenciar de amenazas. |
| **Área de Impacto** | Costo, Cronograma, Alcance, Calidad, Recursos, Técnico, Infraestructura o Reputación. |
| **Probabilidad** | Escala del 1 al 5 (1 = Muy Baja, 5 = Muy Alta). Equivalente a la matriz 5×5. |
| **Puntuación** | Cálculo: `Impacto (1-5) × Probabilidad (1-5)`. Rango de 1 a 25. |

---

## 2. Escala de Impacto para Amenazas (Riesgos Negativos)

| Nivel | Valor | Puntuación Equivalente (× Probabilidad 5) | Descripción |
|:---|:---:|:---:|:---|
| **Muy Bajo** | 1 | 5 | Efecto mínimo en costo, cronograma o entregables. No requiere acción inmediata. |
| **Bajo** | 2 | 10 | Efecto menor pero notable. Se puede aceptar o monitorear. |
| **Medio** | 3 | 15 | Efecto moderado en el costo, cronograma o calidad. Requiere plan de mitigación. |
| **Alto** | 4 | 20 | Aumento significativo de costos o retraso. Acción prioritaria. |
| **Muy Alto (Crítico)** | 5 | 25 | Impacto terminal; el proyecto podría fracasar. Acción inmediata obligatoria. |

**Zonas de priorización:**
- 🟢 **Verde (1-8):** Riesgo Bajo - Aceptar o monitorear
- 🟡 **Amarillo (9-12):** Riesgo Medio - Requiere plan de mitigación
- 🟠 **Naranja (13-16):** Riesgo Alto - Acción prioritaria
- 🔴 **Rojo (17-25):** Riesgo Crítico - Acción inmediata

## 3. Escala de Impacto para Oportunidades (Riesgos Positivos)

| Nivel | Valor | Puntuación Equivalente (× Probabilidad 5) | Descripción |
|:---|:---:|:---:|:---|
| **Muy Bajo** | 1 | 5 | Impacto positivo mínimo; beneficio marginal. |
| **Bajo** | 2 | 10 | Beneficio menor pero aprovechable si hay recursos. |
| **Medio** | 3 | 15 | Impacto sustancial en costos, cronograma o calidad. Vale la pena invertir. |
| **Alto** | 4 | 20 | Podría convertir el proyecto en un éxito destacado. Acción prioritaria. |
| **Muy Alto** | 5 | 25 | Impacto estratégico significativo; explotar agresivamente. |

## 4. Escala de Probabilidad

| Nivel | Valor (1-5) | Descripción | Frecuencia estimada |
|:---|:---:|:---|:---:|
| **Muy Baja** | 1 | Extremadamente improbable que ocurra | < 10% |
| **Baja** | 2 | Improbable, pero posible | 10% - 30% |
| **Media** | 3 | Puede ocurrir en condiciones normales | 30% - 50% |
| **Alta** | 4 | Es probable que ocurra | 50% - 70% |
| **Muy Alta** | 5 | Ocurrirá con certeza o casi certeza | > 70% |

## 6. Otros Conceptos Clave

| Concepto | Definición |
|:---|:---|
| **Detectabilidad** | Indica qué tan fácil es anticipar el evento antes de que ocurra. Escala de 1 a 3: |
| | **1 (Alta):** Se puede ver con suficiente antelación (semanas antes). |
| | **2 (Media):** Se ve justo antes de que ocurra (días antes). |
| | **3 (Baja):** Se detecta mientras sucede o después de ocurrir. |
| **Estado del Riesgo** | **Activo:** El riesgo/oportunidad aún existe y requiere seguimiento. |
| | **Observado:** Se mantiene bajo monitoreo pasivo; no requiere acción inmediata. |
| | **Cerrado:** Ya no representa un factor para el proyecto (se mitigó o materializó). |
| **Estrategia de Respuesta (Amenazas)** | **Mitigar:** Reducir probabilidad o impacto. |
| | **Aceptar:** No tomar acción, asumir la consecuencia. |
| | **Transferir:** Trasladar a tercero (ej. seguros). |
| | **Evitar:** Eliminar la causa del riesgo. |
| **Estrategia de Respuesta (Oportunidades)** | **Explotar:** Asegurar que la oportunidad ocurra. |
| | **Mejorar:** Aumentar probabilidad o impacto positivo. |
| | **Potenciar:** Maximizar el beneficio si ocurre. |
| | **Aceptar:** Aprovechar si ocurre, sin inversión adicional. |
